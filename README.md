# Mogahed OS

Mogahed OS هو Web App عربي RTL لإدارة الحياة والعمل والتركيز والمعرفة وتحليل الحملات الإعلانية.

## المميزات

- App Shell منظم ومقسم إلى ملفات HTML/CSS/JS.
- Vanilla JavaScript بدون React وبدون Backend.
- LocalStorage مع تصدير واستيراد JSON.
- Mobile First وRTL Arabic.
- أقسام: الرئيسية، الأهداف، المشاريع، المهام، المعرفة، الطوارئ، القرارات، المراجعات، الفوز، Dashboard، تحليل الحملات، البحث، النسخ الاحتياطي.
- Dashboard محسوب من البيانات الحقيقية.
- تحليل حملات يحسب الاستثمار، نقطة التعادل، الربح، ROAS، CPA، Margin، المخاطرة وسيناريوهات التسعير.

## التشغيل المحلي

افتح `index.html` مباشرة في المتصفح، أو شغل سيرفر محلي بسيط:

```bash
python -m http.server 8080
```

ثم افتح:

```text
http://localhost:8080
```

## الرفع على GitHub Pages

1. أنشئ Repository جديد باسم `mogahed-os`.
2. ارفع كل الملفات والمجلدات كما هي.
3. من GitHub: Settings → Pages.
4. اختر Branch: `main` وFolder: `/root`.
5. اضغط Save.
6. افتح رابط GitHub Pages بعد ظهوره.

## ملاحظات تقنية

- البيانات تحفظ في LocalStorage على نفس المتصفح.
- يجب تصدير نسخة JSON دوريًا قبل مسح المتصفح أو تغيير الجهاز.
- لا توجد مفاتيح API أو Backend في النسخة الأولى.

## تحديث المعرفة والفيديو V1.1

تم تحسين صفحة المعرفة فقط بدون تغيير باقي الأقسام:

- دعم روابط YouTube Video وPlaylist.
- إضافة حقل YouTube API Key من المزيد ← الإعدادات.
- زر "جلب البيانات" داخل مودال إضافة المعرفة يملأ تلقائيًا: العنوان، النوع، القناة، التصنيف، الوصف/الملخص، مدة الفيديو أو البلاي ليست، وقائمة فيديوهات البلاي ليست.
- تشغيل الفيديو الحالي داخل التطبيق عبر YouTube IFrame Player API.
- تتبع تقدم الفيديو المنفرد.
- تتبع تقدم البلاي ليست بالكامل بناءً على مدة كل فيديو والفيديوهات المكتملة.
- حفظ ملاحظات مرتبطة بتوقيت الفيديو الحالي.
- الضغط على الملاحظة يرجع الفيديو لنفس التوقيت، ولو الملاحظة على فيديو آخر داخل البلاي ليست ينتقل للفيديو ثم للتوقيت.
- عند انتهاء فيديو داخل Playlist ينتقل للفيديو التالي ويحفظ التقدم.

### تفعيل YouTube API

1. افتح التطبيق.
2. ادخل إلى المزيد.
3. اضغط الإعدادات.
4. ضع مفتاح YouTube Data API v3 في خانة YouTube API Key.
5. افتح المعرفة.
6. اضغط إضافة معرفة.
7. ضع رابط فيديو أو Playlist.
8. اضغط جلب البيانات.
9. احفظ أو راجع العنصر الذي تم إنشاؤه.

## V1 Knowledge Learning System

تمت إضافة طبقة تعلم داخل صفحة المعرفة فقط:

- ملخص إحصائي للمعرفة: عدد الدروس، المكتمل، الذي يحتاج مراجعة، الذي تحول لتنفيذ، إجمالي الوقت، ونسبة اكتمال التعلم.
- حالة تعلم لكل فيديو داخل الفيديو نفسه: لم أبدأ، جاري المشاهدة، انتهيت، أحتاج مراجعة، تم تحويله لتنفيذ.
- Tags لكل فيديو، مع فلترة وبحث داخل المعرفة.
- ربط كل فيديو بهدف أو مشروع بشكل مستقل.
- جدولة مراجعة للفيديو بعد 7 أيام.
- تعليم الفيديو كمكتمل مع تحديث التقدم.
- زر "استخرج خطة تنفيذ" يحول الأفعال المستخرجة إلى مهام حقيقية داخل قسم المهام.
- كل هذه البيانات محفوظة داخل `videoContent` لكل فيديو حتى لا تختلط فيديوهات البلاي ليست ببعض.

## V3 — Task System Upgrade

تم تطوير نظام المهام بدون Backend وبدون تغيير بنية المشروع الأساسية.

### المضاف في V3

- Today View أقوى يعرض أولوية اليوم والإجراءات السريعة.
- Week / Open / Done / All Filters.
- Kanban View للحالات: مفتوحة، قيد التنفيذ، مكتملة، مؤجلة.
- Priority Matrix لتقسيم المهام إلى: عاجل ومهم، مهم غير عاجل، سريع، لاحقًا.
- مصدر المهمة: يدوي، معرفة، طوارئ، حملة، مراجعة.
- دعم المهام المتولدة من المعرفة بعلامة مصدر واضحة.
- خطوات فرعية داخل كل مهمة، كل خطوة في سطر.
- إمكانية تعليم كل خطوة كمكتملة.
- عند اكتمال كل الخطوات تتحول المهمة تلقائيًا إلى مكتملة.
- إحصائيات داخل صفحة المهام: اليوم، المفتوح، المكتمل، من المعرفة، المتأخر، اكتمال الخطوات.
- بحث داخل المهام والخطوات والمصدر.
- توافق كامل مع المهام القديمة المخزنة سابقًا.

### اختبار V3

1. افتح صفحة المهام.
2. جرّب الفلاتر: اليوم، الأسبوع، المفتوح، المكتمل، Kanban، Matrix.
3. أضف مهمة جديدة واكتب خطوات فرعية كل واحدة في سطر.
4. اضغط على خطوة فرعية وتأكد أنها تتحول لمكتملة.
5. أكمل كل الخطوات وتأكد أن المهمة تتحول إلى مكتملة.
6. أضف مهمة من المعرفة وتأكد أنها تظهر بمصدر: معرفة.
7. أضف مهمة من الطوارئ وتأكد أنها تظهر بمصدر: طوارئ.
8. استخدم البحث داخل صفحة المهام.

## V4 — Campaign Analyzer Pro

- خطة تنفيذ 3 أو 7 أيام.
