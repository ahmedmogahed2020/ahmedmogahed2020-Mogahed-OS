# نشر Mogahed OS على GitHub Pages

هذا الملف خاص بنسخة **V31 GitHub Pages Deployment Pack**.

## الطريقة المختصرة

1. فك ضغط النسخة.
2. ارفع محتويات المجلد نفسها إلى الريبو، وليس المجلد الخارجي فقط.
3. تأكد أن `index.html` موجود في جذر الريبو.
4. من GitHub افتح: **Settings → Pages**.
5. اختر:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/root`
6. اضغط Save.
7. بعد ظهور رابط GitHub Pages افتحه من الموبايل والكمبيوتر.
8. افتح داخل التطبيق: **المزيد → System Health** ثم شغّل الاختبار.

## مهم جدًا

لا ترفع الملف المضغوط فقط. لازم ترفع الملفات الداخلية نفسها، مثل:

```text
index.html
css/
js/
assets/
README.md
.nojekyll
```

## لماذا يوجد ملف .nojekyll؟

ملف `.nojekyll` يمنع GitHub Pages من التعامل مع المشروع كأنه موقع Jekyll. وجوده آمن ومفيد للمشاريع الثابتة التي تعتمد على ملفات CSS وJS مباشرة.

## اختبار سريع بعد النشر

- افتح الرئيسية.
- افتح المهام.
- افتح المعرفة.
- افتح المزيد.
- افتح النسخ الاحتياطي.
- شغّل System Health.
- تأكد أن تقرير GitHub Pages لا يظهر فيه فشل.

## لو الصفحة فتحت بيضاء

راجع الآتي:

- هل `index.html` في جذر الريبو؟
- هل رفعت مجلدات `css` و `js`؟
- هل GitHub Pages مضبوط على branch الصحيح؟
- افتح Console من المتصفح وشوف هل فيه ملف ناقص 404.

## لو التصميم ظهر بدون تنسيق

غالبًا مجلد `css` لم يُرفع أو اتحط داخل مجلد داخلي بالغلط.

## لو الأزرار لا تعمل

غالبًا مجلد `js` لم يُرفع أو المسار اتغير. لا تغير أسماء المجلدات.
